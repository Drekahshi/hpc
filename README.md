## The Problem

Kenya generates **80% of its electricity from renewables** — one of the highest ratios in the world. Yet its energy sector operates on blind institutional trust:

- 📵 KPLC billing is opaque — consumers cannot verify what they are charged or why
- 🔒 IPP contracts are negotiated in secret — no public pricing, no community input
- 🌳 CFAs and CBOs who steward renewable-rich land receive **zero value** from the energy it generates
- 📊 EPRA cannot regulate effectively without verified, tamper-proof data from the ground up
- 🤖 As AI infrastructure makes energy the new currency of intelligence, this gap becomes a **national equity crisis**

> *"The cost of intelligence will be the cost of energy. AI is dependent on Energy."*
> — Peng Xiao, CEO, G42

The country that controls energy infrastructure will shape AI access. Kenya must build the trust layer now.

---

## The Solution — HISA People Chain

HISA People Chain is a **four-pillar trust layer** built on Hedera Hashgraph that makes Kenya's energy data, community grievances, renewable assets, and cooperative governance **verifiable, transparent, and rewarding** — with HISA operating as the agentic AI intelligence core.

## The Four Pillars

### 🌿 JANI — Energy Asset Verification
**Trust Problem:** Energy assets owned by communities are invisible to the formal system.

JANI anchors CFA and CBO energy assets — solar farms, biogas plants, mini-grids, biomass potential — on Hedera's immutable ledger. Every asset is tokenized, GPS-verified, and assigned a verifiable on-chain credential. This turns informal energy actors into recognized, bankable participants.

```
CFA submits asset → JANI mints HTS credential → Asset anchored on HCS
→ ESG score generated → Green finance eligibility unlocked
```

---

### 💬 CHAT — Community Grievance & Reporting Chain
**Trust Problem:** Billing disputes, outage reports, and corruption demands have no tamper-proof record.

CHAT logs every community energy grievance — outages, bribe demands for rural electrification, billing anomalies — immutably on Hedera Consensus Service. EPRA receives a live, tamper-proof regulatory intelligence feed from the ground up, not filtered through institutional reports.

```
Community submits report → CHAT logs to HCS → Immutable sequence assigned
→ EPRA dashboard updated → Pattern analysis flags corruption hotspots
```

---

### 🤝 UMOJA — Democratic Energy Cooperative Governance
**Trust Problem:** Rural communities have no voice in electrification decisions.

UMOJA enables community energy cooperatives to vote on electrification priorities, energy allocation, and cooperative governance on-chain. Every vote is weighted, recorded, and verifiable — replacing top-down bureaucratic decisions with democratic community will.

```
Cooperative proposes motion → Members vote via UMOJA → Votes recorded on HCS
→ Tally verified by smart contract → Result published to EPRA dashboard
```

---

### ⚡ HISA — Agentic AI Intelligence Layer
**Trust Problem:** No mechanism exists to aggregate community data into actionable policy intelligence — or to route value back to energy producers.

HISA is the agentic AI core that reads all three pillars, generates real-time policy intelligence for EPRA, mints carbon credit tokens via Hedera Guardian, and routes verified value back to the communities who produce Kenya's clean energy. HISA also **advocates for energy's centrality in the AI economy** — making Kenya's communities visible as strategic AI-era energy suppliers.

```
HISA reads JANI + CHAT + UMOJA data → AI analysis run
→ EPRA policy briefing generated → Carbon credits minted
→ Value routed to CFA/CBO wallets → ESG score published
```

## Getting Started

### Prerequisites

```bash
node >= 18.0.0
npm >= 9.0.0
Hedera testnet account (get one at portal.hedera.com)
```

### Installation

```bash
# Clone the repository
git clone https://github.com/Drekahshi/HISA--CHAIN.git
cd HISA--CHAIN

# Install dependencies
npm install

# Copy environment variables
cp .env.example .env
```

### Environment Variables

```env
# Hedera Configuration
HEDERA_ACCOUNT_ID=your_testnet_account_id
HEDERA_PRIVATE_KEY=your_testnet_private_key
HEDERA_NETWORK=testnet

# Pre-deployed Token IDs
JANI_TOKEN_ID=your_jani_token_id
CHAT_TOKEN_ID=your_chat_token_id
UMOJA_TOKEN_ID=your_umoja_token_id
HISA_TOKEN_ID=your_hisa_token_id

# Energy Trust Layer
EPRA_API_KEY=your_epra_api_key
GUARDIAN_API_URL=https://guardian.hedera.com/api
GUARDIAN_POLICY_ID=your_carbon_credit_policy_id
CFA_REGISTRY_ENDPOINT=https://kfs.go.ke/api/cfa-registry

# HISA AI Agent
HISA_AI_MODEL=hisa-agent-v1
```

### Run Development Server

```bash
# Backend
npm run dev:server

# Frontend
npm run dev

# Both concurrently
npm run dev:all
```

## API Endpoints

### JANI — Asset Verification
```
POST /api/jani/verify-asset
Body: { cfa_id, asset_type, gps_coords, capacity_kw }
Returns: { asset_id, hedera_tx_id, verification_hash }
```

### CHAT — Grievance Logger
```
POST /api/chat/log
Body: { community_id, issue_type, location, timestamp }
Returns: { log_id, hcs_sequence, tamper_proof_url }
```

### UMOJA — Governance Vote
```
POST /api/umoja/vote
Body: { cooperative_id, proposal_id, vote, member_id }
Returns: { vote_id, current_tally, quorum_status }
```

### HISA — AI Policy Dashboard
```
GET /api/hisa/policy-dashboard
Returns: { epra_briefing, esg_scores, carbon_credit_yield, alerts }
```
